<h1>Personal To-Do List Application</h1> <h2>Overview of the Project</h2> <p>This is a command-line Python application for managing personal to-do lists. Users can add tasks, view their list, mark tasks as complete, and delete tasks. Each user's tasks are stored in a MySQL database for persistence across sessions. The app prompts for a username at startup and ensures tasks are user-specific. It's designed for simplicity and efficiency, with basic error handling for database operations.</p> <h2>Features</h2> <ul> <li><strong>User Personalization</strong>: Enter a name to access or create a personalized to-do list.</li> <li><strong>Add Tasks</strong>: Add new tasks with custom descriptions.</li> <li><strong>View Tasks</strong>: Display all tasks for the current user, sorted by status and ID, with visual indicators for completion.</li> <li><strong>Mark Tasks Complete</strong>: Update a task's status to "Complete" by providing the exact description.</li> <li><strong>Delete Tasks</strong>: Remove tasks by providing the exact description.</li> <li><strong>Database Persistence</strong>: Store tasks in MySQL, with automatic database and table creation.</li> <li><strong>Secure Password Handling</strong>: Prompts for MySQL password securely using getpass.</li> <li><strong>User-Friendly Menu</strong>: Simple CLI interface with options to manage tasks or exit.</li> </ul> <h2>Technologies/Tools Used</h2> <ul> <li><strong>Programming Language</strong>: Python 3.x</li> <li><strong>Database</strong>: MySQL (for data storage and retrieval)</li> <li><strong>Libraries</strong>: <ul> <li><code>mysql-connector-python</code>: For MySQL database connections.</li> <li><code>getpass</code>, <code>contextlib</code>, <code>sys</code>: Built-in Python modules for password input, connection management, and system operations.</li> </ul> </li> <li><strong>Other Tools</strong>: Command-line interface (no GUI required).</li> </ul> <h2>Steps to Install & Run the Project</h2> <h3>Prerequisites</h3> <ul> <li>Python 3.x installed on your system.</li> <li>MySQL Server installed and running (e.g., via XAMPP, WAMP, or native installation).</li> <li>A MySQL user with privileges to create databases (default assumed: 'root').</li> </ul> <h3>Installation Steps</h3> <ol> <li><strong>Clone or Download the Code</strong>: <ul> <li>Download the script file (e.g., <code>todo_list.py</code>) to your local machine.</li> </ul> </li> <li><strong>Install Required Python Libraries</strong>: <ul> <li>Open a terminal/command prompt and run:</li> </ul> <pre><code>pip install mysql-connector-python</code></pre> <ul> <li>If you encounter issues, ensure <code>pip</code> is up-to-date: <code>pip install --upgrade pip</code>.</li> </ul> </li> <li><strong>Set Up MySQL</strong>: <ul> <li>Start your MySQL server.</li> <li>Ensure the 'root' user exists and has database creation permissions. If needed, create a new user via MySQL CLI:</li> </ul> <pre><code>CREATE USER 'root'@'localhost' IDENTIFIED BY 'your_password'; GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost'; FLUSH PRIVILEGES;</code></pre> </li> <li><strong>Run the Script</strong>: <ul> <li>Navigate to the directory containing <code>todo_list.py</code>.</li> <li>Execute the script:</li> </ul> <pre><code>python todo_list.py</code></pre> <ul> <li>On first run, enter your MySQL root password when prompted. The app will automatically create the database (<code>todo_list_db</code>) and table (<code>tasks</code>).</li> </ul> </li> <li><strong>Start Using the App</strong>: <ul> <li>Follow the on-screen menu to select options (e.g., add a task, view list).</li> </ul> </li> </ol> <h3>Troubleshooting</h3> <ul> <li><strong>Connection Errors</strong>: Verify MySQL is running and credentials are correct. Check firewall settings.</li> <li><strong>Library Issues</strong>: Ensure <code>mysql-connector-python</code> is installed correctly.</li> <li><strong>Permissions</strong>: Ensure the MySQL user can create databases.</li> </ul> <h2>Instructions for Testing</h2> <ol> <li><strong>Basic Functionality Test</strong>: <ul> <li>Run the app and enter a username (e.g., "Alice").</li> <li>Select "2. Add New Task" and enter a description (e.g., "Buy groceries").</li> <li>Select "1. View My Tasks" to verify the task appears.</li> <li>Check the database: Log into MySQL and query <code>SELECT * FROM todo_list_db.tasks WHERE user_name='Alice';</code> to confirm data is saved.</li> </ul> </li> <li><strong>Mark Complete Test</strong>: <ul> <li>Add a task as above.</li> <li>Select "3. Mark Task Complete" and enter the exact description.</li> <li>View tasks again to confirm status changes to "Complete" with [X].</li> </ul> </li> <li><strong>Delete Task Test</strong>: <ul> <li>Add a task.</li> <li>Select "4. Delete Task" and enter the exact description.</li> <li>View tasks to confirm removal.</li> </ul> </li> <li><strong>Edge Case Testing</strong>: <ul> <li>Test with empty inputs – should prompt for valid descriptions.</li> <li>Try marking or deleting non-existent tasks – should show appropriate messages.</li> <li>Simulate DB disconnection (stop MySQL) and verify error handling.</li> <li>Use tools like <code>unittest</code> for automated tests (e.g., create a test file to mock DB calls and validate task operations).</li> </ul> </li> <li><strong>Performance Test</strong>: <ul> <li>Run multiple sessions with different users to ensure DB queries are efficient.</li> </ul> </li> </ol> <p>For automated testing, consider adding unit tests to the script (e.g., using Python's <code>unittest</code> module to test functions like <code>add_task</code>).</p> <h2>Screenshots</h2> <p><em>(Screenshots are optional but recommended; add them here if available.
